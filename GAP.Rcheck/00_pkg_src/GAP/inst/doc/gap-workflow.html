GAP workflow with packaged chr22 example data

GAP fits a genetic association with pleiotropy model using summary-level effect sizes and standard errors.

This vignette shows how to:

  1. Load the packaged example files.
  2. Parse SNP, chromosome, and position fields from the chr22 example IDs.
  3. Build a GAP_bayesian_prior() input table.
  4. Run a small prior-estimation example.

Build a GAP input table

The example IDs include chromosome and genomic position in the form gene-chr:pos_ref/alt. The helper code below extracts the pieces needed by the GAP functions.

For illustration, we use:

  • Astrocytes as beta_01
  • Excitatory.neurons as beta_12
  • bulk as beta_cc

You can swap these columns for the pair of cell types relevant to your analysis.

parse_chr22_id <- function(x) {
  chr_pos <- sub("^.*-([0-9]+):([0-9]+)_.*$", "\\1\t\\2", x)
  parts <- strsplit(chr_pos, "\t", fixed = TRUE)
  data.frame(
    chr = as.integer(vapply(parts, `[`, "", 1)),
    pos = as.integer(vapply(parts, `[`, "", 2))
  )
}

loc <- parse_chr22_id(beta$ID)

gap_input <- data.frame(
  snp = beta$ID,
  chr = loc$chr,
  pos = loc$pos,
  beta_01 = beta$Astrocytes,
  beta_12 = beta$Excitatory.neurons,
  beta_cc = beta$bulk,
  se_01 = se$Astrocytes,
  se_12 = se$Excitatory.neurons,
  se_cc = se$bulk,
  check.names = FALSE
)

Estimate a prior on a small subset

GAP_bayesian_prior() can be expensive on large inputs, so the vignette uses a small subset just to show the workflow.

set.seed(1)
subset_idx <- sample(seq_len(nrow(gap_input)), 250)
prior_example <- GAP_bayesian_prior(
  input = gap_input[subset_idx, ],
  alpha = 0,
  p_threshold = 1e-4,
  random = 100
)
#> Check chr  22 
#> 12  significant snps in chr  22 
#> Check chr  22

After estimating the prior, you can pass it to GAP_bayesian_lrt() on the dataset you want to analyze.

lrt_results <- GAP_bayesian_lrt(
  input = gap_input,
  alpha = 0,
  prior = prior_example
)
